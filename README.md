## 運用ルール

### 振り返りの観点

各活動の振り返りでは以下の観点で整理する。

1. **業務種別（Category）** — どの業務に関する課題か
2. **事実（Fact）** — 何が起きたか
3. **課題（Problem）** — 何が問題だったか
4. **原因（Cause）** — なぜ起きたか
5. **対策（Action）** — 次にどうするか
6. **仮説（Hypothesis）** — 次の打ち手として何を試すか

### 更新頻度

- 各活動の振り返りは **週次** または **スプリント単位** で実施
- 大きな課題やインシデントが発生した場合は都度追記

## 影響度の定義

| レベル | 定義 |
|--------|------|
| 高 | プロジェクト全体の進行・品質に重大な影響 |
| 中 | 特定の工程やチームに影響 |
| 低 | 軽微な影響、改善すればベター |

## ステータスの定義

| ステータス | 定義 |
|-----------|------|
| 未着手 | まだ対応を開始していない |
| 進行中 | 対応を進めている |
| 完了 | 対応が完了し効果を確認済み |
| 保留 | 優先度や状況により一時保留 |

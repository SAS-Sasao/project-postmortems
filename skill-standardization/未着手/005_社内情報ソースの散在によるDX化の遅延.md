# 005: 社内情報ソースの散在によるDX化の遅延

- **日付**: 2026-03-08
- **業務種別**: 方法論策定（ID: K08）
- **事実（Fact）**: 取り扱う情報ソースがSharePoint、desknet's、ローカルのExcelファイル、GitHubなどにまちまちに保存されており、必要な情報を探すのに工数がかかっている
- **課題（Problem）**: 情報の散在により業務効率が低下し、社内のDX化が遅れている
- **原因（Cause）**: 情報ソースの統合・移行をリードする専任担当者が不在で、かつ移行の目的・恩恵が社内に共有されていないため、全社的な取り組みとして推進できていなかった
- **影響度**: 高（ID: H）
- **ステータス**: 未着手（ID: S01）

## 改善アクション

| # | アクション | 期限 | ステータス |
|---|-----------|------|-----------|
| 1 | 情報ソース移行を専属でリードするメンバーを1名以上アサインする | - | 未着手 |
| 2 | 現在の情報ソースの棚卸しを行い、移行先の統合プラットフォームを選定する（下記参照） | - | 未着手 |
| 3 | 「何故情報ソースを移すのか」「移した場合の恩恵」を明文化し、社内全体に周知する説明資料を作成する | - | 未着手 |
| 4 | 移行スケジュールと段階的な移行計画を策定し、社員が無理なく対応できる仕組みを作る | - | 未着手 |

### 現状の情報ソース棚卸し・移行先検討の例

| # | 現在の情報ソース | 主な格納情報 | 移行先候補 |
|---|-----------------|-------------|-----------|
| 1 | SharePoint | 社内文書、プロジェクト資料 | SharePoint（統合・整理） |
| 2 | desknet's | スケジュール、掲示板、ワークフロー | SharePoint / Microsoft 365 |
| 3 | ローカルExcelファイル | 管理表、一覧、集計データ | SharePoint / クラウドDB |
| 4 | GitHub | ソースコード、技術ドキュメント | GitHub（整理・権限統一） |

## 次の打ち手（仮説）

- 専任リーダーをアサインし移行の恩恵を明確に共有することで、社員1人1人が「自分ごと」として移行に協力する体制を作れるのではないか
- 情報ソースを統合することで、情報検索の工数を大幅に削減し、業務効率の向上と社内DX化の加速に繋がるのではないか

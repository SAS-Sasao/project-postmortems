# 009: ツール・アカウント発行管理の未統一

- **日付**: 2026-03-09
- **業務種別**: 方法論策定（ID: K08）
- **事実（Fact）**: 受託案件で使用するツールやアカウントの発行・管理について、担当者やルールが統一されていない。AWSアカウント発行は現場のインフラ担当、GitHubアカウント発行は一部知見がある方、SharePoint発行は社内DX担当、Backlogは偉い人または社内DX担当が対応しており、バラバラな運用になっている
- **課題（Problem）**: アカウント発行・ツール管理の担当者とルールが散在しており、案件立ち上げ時の対応が属人的かつ非効率になっている
- **原因（Cause）**: 受託案件で発生するアカウント発行やツール利用に関する一覧が整理されておらず、誰がどのツールの管理権限を持つかのルールが策定されていなかった
- **影響度**: 高（ID: H）
- **ステータス**: 未着手（ID: S01）

## 改善アクション

| # | アクション | 期限 | ステータス |
|---|-----------|------|-----------|
| 1 | 受託案件で発生する全ツール・アカウント発行の一覧を作成し、現在の担当者・発行手順・管理権限を整理する | - | 未着手 |
| 2 | アカウント発行・管理権限の集約先（社内DX担当）への移管計画を策定し、優先順位を決定する | - | 未着手 |
| 3 | ツール・アカウント発行の申請フローと管理ルール（申請先・承認者・手順）を標準化したドキュメントを作成する | - | 未着手 |
| 4 | 案件立ち上げ時のツール・アカウント発行チェックリストを作成し、運用ルールとして定着させる | - | 未着手 |

## 次の打ち手（仮説）

- アカウント発行・管理権限を社内DX担当に集約することで、属人化を解消し、案件立ち上げ時の対応速度と正確性を向上できるのではないか
- 標準化された申請フローとチェックリストを整備することで、新規案件のたびに「誰に聞けばいいか分からない」という問題を防げるのではないか

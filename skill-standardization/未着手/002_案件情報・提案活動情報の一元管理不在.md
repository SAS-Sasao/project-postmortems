# 002: 案件情報・提案活動情報の一元管理不在

- **日付**: 2026-03-07
- **業務種別**: 方法論策定（ID: K08）
- **事実（Fact）**: 社内で過去・現在対応した案件情報や提案活動の情報が一元管理されておらず、参考にできる資料がどこにあるのか不明瞭な状態にある
- **課題（Problem）**: 過去の実績・知見を活用できず、同じ失敗の繰り返しや提案品質のばらつきが発生している
- **原因（Cause）**: 案件情報を集約・管理するためのツール・運用ルール・責任者が定められていなかった
- **影響度**: 高（ID: H）
- **ステータス**: 未着手（ID: S01）

## 改善アクション

| # | アクション | 期限 | ステータス |
|---|-----------|------|-----------|
| 1 | 案件情報を一元管理するプラットフォーム（SharePoint等）を選定・構築する | - | 未着手 |
| 2 | 案件情報の登録項目（下記参照）を定義し、全案件で統一フォーマットで記録する運用ルールを策定する | - | 未着手 |
| 3 | 既存の案件情報を棚卸しし、一元管理プラットフォームへ移行する | - | 未着手 |

### 案件情報の登録項目例

| # | カテゴリ | 登録項目 |
|---|---------|---------|
| 1 | 基本情報 | 案件名、顧客名、業界、対応期間、ステータス |
| 2 | 体制情報 | ステークホルダー、自社体制（役割・人数）、パートナー |
| 3 | 技術情報 | 対象アプリケーション、技術スタック、アーキテクチャ |
| 4 | 提案情報 | 提案書、見積書、プレゼン資料のリンク |
| 5 | 振り返り | 成功要因、課題、教訓 |

## 次の打ち手（仮説）

- 案件情報を一元管理することで、類似案件の提案時に過去の実績を即座に参照でき、提案品質と提案スピードを向上できるのではないか
- 将来的にデータレイク・データウェアハウスに格納し分析基盤を構築することで、営業レポートやマーケティング活動に活用できるデータドリブンな経営判断が可能になるのではないか

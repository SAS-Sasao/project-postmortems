# 003: 標準化ドキュメントの形骸化

- **日付**: 2026-03-07
- **業務種別**: 方法論策定（ID: K08）
- **事実（Fact）**: スキル標準化のために過去に作成したドキュメントの最終更新が20年前のものがあり、現在の開発手法（AI駆動開発、アジャイル開発等）に対応していない
- **課題（Problem）**: 形骸化したドキュメントが実務で参照されず、標準化の基盤として機能していない
- **原因（Cause）**: ドキュメントの定期メンテナンス体制・責任者・更新サイクルが確立されていなかった
- **影響度**: 高（ID: H）
- **ステータス**: 未着手（ID: S01）

## 改善アクション

| # | アクション | 期限 | ステータス |
|---|-----------|------|-----------|
| 1 | 既存ドキュメントを棚卸しし、更新・廃止・新規作成の判定を行う | - | 未着手 |
| 2 | 現代の開発手法（AI駆動開発、アジャイル、モダンなウォーターフォール）に対応した内容にドキュメントを刷新する | - | 未着手 |
| 3 | ドキュメントメンテナンスチームを編成し、半年〜1年周期での定期レビュー・更新サイクルを確立する | - | 未着手 |
| 4 | メンテナンスメンバーは技術への関心が高く情報発信力のあるメンバーを選定する基準を設ける | - | 未着手 |

## 次の打ち手（仮説）

- ドキュメントを現代の開発手法に刷新することで、実務で実際に参照・活用されるドキュメントとなり、標準化の実効性を回復できるのではないか
- 定期メンテナンスサイクルを確立し技術志向の高いメンバーをアサインすることで、ドキュメントの鮮度を維持し再度の形骸化を防げるのではないか

# 008: ドメイン知識のナレッジ不足

- **日付**: 2026-03-08
- **業務種別**: 方法論策定（ID: K08）
- **事実（Fact）**: スキル標準化において、業界・業務知識（ドメイン知識）をシェアするドキュメントが不足しており、個人の経験に依存している
- **課題（Problem）**: AI駆動開発時代においてSI事業の差別化要因となるドメイン知識が組織のナレッジとして蓄積・活用できていない
- **原因（Cause）**: ドメイン知識を体系的に整理・ドキュメント化するプロセスと、それを管理・共有する基盤が存在しなかった
- **影響度**: 高（ID: H）
- **ステータス**: 未着手（ID: S01）

## 改善アクション

| # | アクション | 期限 | ステータス |
|---|-----------|------|-----------|
| 1 | ドメイン知識ドキュメントの体系（下記参照）を定義し、業界別・業務別にナレッジを整理するフォーマットを作成する | - | 未着手 |
| 2 | 案件完了時に得たドメイン知識をドキュメント化するプロセスを案件クローズ手順に組み込む | - | 未着手 |
| 3 | ドメイン知識をAI駆動開発で活用できる形式（ルールファイル・スキルファイル等）に変換するための準備を行う | - | 未着手 |

### ドメイン知識ドキュメントの体系例

| # | カテゴリ | 内容 |
|---|---------|------|
| 1 | 業界概要 | 対象業界の構造・トレンド・主要プレイヤー |
| 2 | 業務フロー | 主要な業務プロセスとその流れ |
| 3 | 業界用語集 | 業界特有の用語と定義 |
| 4 | 法規制・コンプライアンス | 業界固有の規制や遵守事項 |
| 5 | システム要件の傾向 | 業界特有のシステム要件・非機能要件 |
| 6 | 過去の案件実績 | 当該業界での自社の対応実績・教訓 |

## 次の打ち手（仮説）

- ドメイン知識をドキュメント化し体系的に管理することで、新規案件参画時の立ち上がりスピードが向上し、提案品質の向上にも繋がるのではないか
- 将来的にドメイン知識をデータレイク・DWHに格納しAI駆動開発のルール・スキルファイルとして活用することで、AIが業界固有の要件を理解した上でコード生成でき、開発品質と効率が飛躍的に向上するのではないか

# 社内スキル標準化活動 - マスタ定義

課題の振り分け・分類に使用するマスタデータ。コマンド実行時に本ファイルを参照して自動分類を行う。

---

## 業務種別

| ID | 業務種別 | 説明 | 分類キーワード |
|----|----------|------|---------------|
| K01 | 提案 | 受託案件獲得のための提案書作成・プレゼン・見積り | 提案, 提案書, プレゼン, 見積り, RFP, コンペ, 営業 |
| K02 | デザイン | UI/UXデザイン・画面デザイン・デザインガイドライン | デザイン, UI, UX, 画面, ワイヤーフレーム, モックアップ |
| K03 | アーキテクチャ | システム構成・技術選定・非機能要件の設計方針 | アーキテクチャ, 構成, 技術選定, 非機能, インフラ設計 |
| K04 | 開発 | コーディング規約・開発プロセス・レビュー基準 | 開発, コーディング, 規約, プロセス, レビュー, Git |
| K05 | テスト | テスト方針・テスト計画・品質基準の標準化 | テスト, 品質, 基準, 計画, 自動テスト, カバレッジ |
| K06 | 移行 | データ移行手順・環境構築手順の標準化 | 移行, データ移行, 環境構築, 手順書, チェックリスト |
| K07 | リリース | リリース判定基準・リリース手順の標準化 | リリース, 判定, 手順, チェックリスト, 承認フロー |
| K08 | 方法論策定 | 上記横断の方法論・プロセス全体の策定と改善 | 方法論, プロセス, 標準化, テンプレート, ガイドライン, フレームワーク |

## 影響度

| ID | レベル | 定義 | 分類キーワード |
|----|--------|------|---------------|
| H | 高 | 標準化活動全体の進行・社内展開に重大な影響 | 致命的, 重大, 全体, 展開不可, ブロッカー, 停止 |
| M | 中 | 特定のフェーズや成果物に影響があるが活動全体は継続可能 | 一部, 特定, 遅れ, 手戻り, 修正 |
| L | 低 | 軽微な影響、改善すればベター | 軽微, 小さい, 改善, ベター, 気になる |

## ステータス

| ID | ステータス | 定義 |
|----|-----------|------|
| S01 | 未着手 | まだ対応を開始していない |
| S02 | 進行中 | 対応を進めている |
| S03 | 完了 | 対応が完了し効果を確認済み |
| S04 | 保留 | 優先度や状況により一時保留 |

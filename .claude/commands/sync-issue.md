対象: $ARGUMENTS

---

以下の手順でGitHub Issueの状態をローカルの課題ファイル・TODOファイルに同期せよ。全手順においてCLAUDE.mdのルールを絶対遵守すること。

## 手順

### Step 1: GitHub Issueの情報取得

$ARGUMENTS からIssue番号を特定し、`gh issue view` でIssueの情報を取得する。

```bash
gh issue view <Issue番号> --json title,body,state
```

取得する情報:
- **title**: Issueタイトル（活動名・課題番号・課題タイトルの特定に使用）
- **body**: Issue本文（チェックリストの状態確認に使用）
- **state**: Issueの状態（OPEN / CLOSED）

Issueが存在しない場合はユーザーに報告して終了する。

### Step 2: 対象ファイルの特定

Issueタイトルのフォーマット `[活動名] NNN: 課題タイトル` から以下を特定する。

1. **活動ディレクトリ**:
   - `[DJNS]` → djns/
   - `[AI推進]` → ai-promotion/
   - `[スキル標準化]` → skill-standardization/

2. **課題番号**: タイトル内の `NNN` 部分（例: `004`）

3. **対象ファイルの探索**: `進行中/` ディレクトリ内で課題番号に合致する課題ファイル（`NNN_*.md`）とTODOファイル（`NNN_*_todo.md`）を特定する

対象ファイルが見つからない場合はユーザーに報告し、確認を取る。

### Step 3: チェックリストの差分解析

Issue本文のチェックリストとTODOファイルのタスク一覧を比較する。

**チェックリストの解析ルール**:
- `- [x]` → チェック済み（完了）
- `- [ ]` → 未チェック（未完了）

**マッチングルール**:
- Issue本文のチェックリスト項目を上から順に1, 2, 3...と番号付けする
- TODOファイルのタスク一覧の `#` 列と対応させる
- チェック済みの項目に対応するTODOタスクのステータスを `完了` に更新する
- 既に `完了` のタスクはそのまま維持する

### Step 4: 変更内容のフィードバック確認

解析結果をユーザーに提示し、確認を取る。

**Issueがオープンの場合**:

```
【同期内容】

- GitHub Issue: #N (OPEN)
- 課題ファイル: NNN_課題タイトル.md
- TODOファイル: NNN_課題タイトル_todo.md

タスク進捗:
- [完了] タスク1: xxx（新規完了 ✓）
- [未着手] タスク2: xxx
- [完了] タスク3: xxx（変更なし）
...

進捗: N/M タスク完了

この内容でTODOファイルを更新しますか？
```

**Issueがクローズの場合**:

```
【同期内容】

- GitHub Issue: #N (CLOSED)
- 課題ファイル: NNN_課題タイトル.md
- TODOファイル: NNN_課題タイトル_todo.md

Issueがクローズされています。以下の操作を行います:

1. TODOファイルの全タスクを「完了」に更新
2. 課題ファイルのステータスを「完了（ID: S03）」に更新
3. 課題ファイルとTODOファイルを「進行中/」→「完了/」へ移動
4. README.mdのリンクとステータスを更新

この内容で同期しますか？
```

ユーザーの承認後、次のステップへ進む。

### Step 5: TODOファイルの更新

TODOファイルのタスク一覧テーブルを更新する。

**更新ルール**:
- チェック済みタスク: `| N | タスク内容 | - | 未着手 |` → `| N | タスク内容 | - | 完了 |`
- 未チェックタスク: 変更なし

### Step 6: Issueクローズ時の追加処理

Issueの状態が `CLOSED` の場合、以下の追加処理を行う。

1. **課題ファイル内のステータスを更新**: `進行中（ID: S02）` → `完了（ID: S03）`
2. **ファイルを `進行中/` から `完了/` へ移動**: `git mv` を使用（課題ファイルとTODOファイルの両方）
3. **README.mdのリンクを更新**: 対象ディレクトリのREADME.mdとルートREADME.mdの課題テーブルのリンクパスとステータスを更新
   - リンクパス: `進行中/NNN_*.md` → `完了/NNN_*.md`
   - ステータス列: `進行中` → `完了`

### Step 7: 整合性チェック

以下を確認する。

**共通チェック**:
- TODOファイルのタスクステータスがIssueのチェックリストと一致しているか

**Issueクローズ時の追加チェック**:
- 課題ファイルとTODOファイルが `完了/` ディレクトリに正しく移動されているか
- 課題ファイル内のステータスが `完了（ID: S03）` に更新されているか
- README.mdのリンクが `完了/` パスに更新されているか

問題があればユーザーに報告し、修正の確認を取る。

### Step 8: 完了報告

**Issueオープン時**:

```
【完了】

- GitHub Issue: #N
- TODOファイル: NNN_課題タイトル_todo.md
- 更新タスク: N件のタスクを「完了」に更新
- 進捗: N/M タスク完了

コミット・プッシュしますか？
```

**Issueクローズ時**:

```
【完了】

- GitHub Issue: #N (CLOSED)
- 課題ファイル: NNN_課題タイトル.md
- TODOファイル: NNN_課題タイトル_todo.md
- ステータス: 進行中 → 完了
- 格納先: 進行中/ → 完了/

コミット・プッシュしますか？
```

以下の整合性チェックをリポジトリ全体に対して実施せよ。全手順においてCLAUDE.mdのルールを絶対遵守すること。

## チェック項目

### 1. 課題ファイルのナンバリング重複チェック

各活動ディレクトリ（djns/, ai-promotion/, skill-standardization/）内の全ステータスディレクトリ（未着手/ 進行中/ 完了/ 保留/）を横断して課題mdファイル（NNN_*.md）を走査し、同一活動ディレクトリ内でナンバリングが重複していないか確認する。

### 2. リンク整合性チェック

リポジトリ内の全mdファイルを走査し、以下を確認する。

- ファイル内リンク（`[text](path)`形式）のリンク先が実在するか
- アンカーリンク（`#`形式）の参照先が存在するか
- 相対パスが正しく解決できるか

### 3. README.mdテーブルと実ファイルの整合性

各活動ディレクトリのREADME.mdの課題テーブルについて以下を確認する。

- テーブルに記載された課題ファイルが実在するか
- 実在する課題ファイルがテーブルに漏れなく記載されているか
- ルートREADME.mdと各ディレクトリREADME.mdの間で情報の不整合がないか

### 4. テンプレート準拠チェック

各課題mdファイルが所定のテンプレート構造に従っているか確認する。

必須項目:
- 日付
- 業務種別
- 事実（Fact）
- 課題（Problem）
- 原因（Cause）
- 影響度

### 5. ステータスディレクトリ整合性チェック

課題ファイルの格納先ディレクトリとファイル内のステータス表記が一致しているか確認する。

- `未着手/` に格納されたファイル → ステータスが「未着手（ID: S01）」であること
- `進行中/` に格納されたファイル → ステータスが「進行中（ID: S02）」であること
- `完了/` に格納されたファイル → ステータスが「完了（ID: S03）」であること
- `保留/` に格納されたファイル → ステータスが「保留（ID: S04）」であること

### 6. マスタ整合性チェック

各活動ディレクトリの `master.md` を読み込み、課題ファイルの内容と照合する。

- 課題ファイル内の業務種別IDが master.md に定義されているか
- 課題ファイル内の影響度IDが master.md に定義されているか
- 課題ファイル内のステータスが master.md に定義されているか
- master.md 自体のID重複がないか

## 結果報告

チェック結果を以下の形式で報告する。

```
## 整合性チェック結果

- チェック日時: YYYY-MM-DD
- 対象ファイル数: N件

### 問題なし
- [問題がなかった項目を列挙]

### 要対応
- [問題があった項目と内容を列挙]
```

問題が検出された場合はユーザーに修正内容を提示し、確認を取ってから修正を実施する。
問題がなければその旨を報告して完了とする。

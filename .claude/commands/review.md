対象活動: $ARGUMENTS

---

以下の手順で振り返りを実施せよ。全手順においてCLAUDE.mdのルールを絶対遵守すること。

## 絶対原則

- 必ずagent-teams（TeamCreate）を使用してチームを構成し、複数エージェントで並列処理すること
- サブエージェントにy/n確認は求めず、直接作業を実施して結果を返すよう指示すること
- サブエージェントは必ず既存の課題ファイル・README.mdを調査すること

## 対象活動の特定とマスタ読み込み

$ARGUMENTS から対象を判定する。

- "djns" または "DJNS" → djns/
- "ai" または "AI推進" → ai-promotion/
- "skill" または "スキル標準化" → skill-standardization/
- "all" または "全体" → 全ディレクトリ対象

対象が不明な場合はユーザーに確認を取る。

**特定後、対象ディレクトリの `master.md` を必ず読み込む。**（"all"の場合は全ディレクトリの master.md を読み込む）
以降のステップでは master.md に定義された業務種別・影響度・ステータスのみを使用すること。

## チーム構成

以下の3エージェント構成でチームを作成してください。

### 1. issue-scanner（課題・アクション現状調査担当）

- **subagent_type**: general-purpose
- **役割**: 対象活動の課題ファイルと改善アクションの現状を調査
- **作業内容**:
  - **最初に対象ディレクトリの master.md を読み込み、業務種別・影響度・ステータスの定義を把握する**
  - 対象ディレクトリ内の全課題mdファイル（NNN_*.md）を走査
  - 各課題の内容（タイトル・業務種別ID・影響度ID・ステータス）を一覧化
  - 改善アクションのステータス別集計（master.md のステータス定義に準拠）
  - 期限切れのアクションを特定
  - 「次の打ち手（仮説）」の現状を収集
  - 調査結果をチームリーダーにSendMessageで報告

### 2. trend-analyzer（傾向分析担当）

- **subagent_type**: general-purpose
- **役割**: 課題の傾向やパターンを分析
- **作業内容**:
  - **最初に対象ディレクトリの master.md を読み込み、業務種別・影響度の全定義を把握する**
  - 対象ディレクトリ内の全課題mdファイルを走査
  - master.md の業務種別ID別に課題数を集計（IDが未使用の種別も0件として報告）
  - master.md の影響度ID別に課題数を集計
  - 繰り返し発生している課題パターンがあれば特定
  - 改善アクションの完了率を算出
  - 分析結果をチームリーダーにSendMessageで報告

### 3. updater（振り返り結果反映担当）

- **subagent_type**: general-purpose
- **役割**: Phase 1の結果とユーザーヒアリングの内容をファイルに反映
- **作業内容**:
  - **最初に対象ディレクトリの master.md を読み込む**
  - チームリーダーから共有された更新内容をファイルに反映
  - 既存課題のステータス更新（master.md のステータスIDに準拠）
  - 新規課題がある場合:
    - ユーザーの文章入力を master.md の分類キーワードと照合し、業務種別・影響度を自動判定
    - 自動判定結果はチームリーダー経由でユーザーに確認を取ること
    - テンプレートに沿ってmdファイルを作成（ナンバリング採番含む、業務種別ID・影響度IDを付与）
  - 各README.mdのテーブルを更新
  - 更新結果をチームリーダーにSendMessageで報告

## 実行フロー

```
Phase 1: 現状調査（並列実行）
  issue-scanner → 課題・アクションの現状一覧化
  trend-analyzer → 傾向分析

Phase 2: ユーザーヒアリング（チームリーダーが実施）
  Phase 1の結果をユーザーに提示し、以下を順番にヒアリング:
    a. 既存課題・アクションのステータス更新はあるか
    b. 新たな課題はあるか
    c. 次の打ち手（仮説）の追加・更新はあるか

Phase 3: 反映（Phase 2完了後）
  updater → ヒアリング結果をファイルに反映

Phase 4: 整合性チェック（チームリーダーが実施）
  リポジトリ全体のナンバリング重複・リンク切れチェック
```

## タスク管理

TaskCreate/TaskUpdate/TaskListを使用してタスクの進捗を管理してください：
1. 各エージェントの作業をTaskCreateで登録
2. 依存関係をaddBlockedByで設定（Phase 3はPhase 2完了後）
3. 作業開始時にin_progress、完了時にcompletedに更新

## 作業指示の共通事項

全エージェントに以下を指示してください：
- y/n確認は一切不要、直接作業を実施すること
- 作業完了後はチームリーダーにSendMessageで報告すること
- 日本語で回答すること

## あなた（チームリーダー）のタスク

1. TeamCreateでチーム作成
2. TaskCreateでタスク登録・依存関係設定
3. Phase 1: issue-scannerとtrend-analyzerを**並列起動**
4. Phase 1完了後: 結果をユーザーに提示しヒアリング実施
5. Phase 2完了後: ヒアリング結果をupdaterに共有してupdaterを起動
6. Phase 3完了後: 整合性チェックを実施
7. 結果を報告し、コミット・プッシュの確認を取る
8. チーム解散（TeamDelete）

以下の手順で全体ステータスを表示せよ。全手順においてCLAUDE.mdのルールを絶対遵守すること。

## 絶対原則

- 必ずagent-teams（TeamCreate）を使用してチームを構成し、複数エージェントで並列処理すること
- サブエージェントにy/n確認は求めず、直接作業を実施して結果を返すよう指示すること
- サブエージェントは必ず既存の課題ファイル・README.mdを調査すること

## チーム構成

以下の4エージェント構成でチームを作成してください。

### 1. djns-scanner（DJNS調査担当）

- **subagent_type**: general-purpose
- **役割**: DJNSプロジェクトの現状を調査
- **作業内容**:
  - **最初に djns/master.md を読み込み、業務種別・影響度・ステータスの定義を把握する**
  - djns/ 配下の全ステータスディレクトリ（未着手/ 進行中/ 完了/ 保留/）を走査し、各ディレクトリ内の課題mdファイル（NNN_*.md）を収集
  - 各課題の内容（タイトル・業務種別ID・影響度ID・格納先ステータスディレクトリ）を一覧化
  - ステータスディレクトリ別の課題数を集計
  - 期限切れのアクションを特定し警告フラグを付与
  - 「次の打ち手（仮説）」の現状を収集
  - 調査結果をチームリーダーにSendMessageで報告

### 4. reporter（統合レポート作成担当）

- **subagent_type**: general-purpose
- **役割**: 全エージェントの調査結果を統合し、ステータスレポートを作成
- **作業内容**:
  - djns-scanner, ai-scanner, skill-scannerの結果を統合
  - 以下の構造でレポートを作成:

```markdown
## 全体ステータスレポート（YYYY-MM-DD）

### サマリー

| 活動 | 課題数 | 未着手 | 進行中 | 完了 | 保留 | 期限切れ |
|------|--------|--------|--------|------|------|----------|

### 活動別詳細

#### DJNS
- 課題一覧（業務種別・影響度付き）
- 期限切れアクション（該当があれば警告表示）

#### AI推進
- 課題一覧（業務種別・影響度付き）
- 期限切れアクション（該当があれば警告表示）

#### スキル標準化
- 課題一覧（業務種別・影響度付き）
- 期限切れアクション（該当があれば警告表示）

### 横断分析
- 影響度「高」の未完了課題一覧
- 全活動共通で見られる傾向
```

  - レポートをチームリーダーにSendMessageで報告

## 実行フロー

```
Phase 1: 調査（並列実行）
  djns-scanner → DJNSの現状調査
  ai-scanner → AI推進の現状調査
  skill-scanner → スキル標準化の現状調査

Phase 2: 統合レポート作成（Phase 1完了後）
  reporter → 結果統合・レポート作成
```

## タスク管理

TaskCreate/TaskUpdate/TaskListを使用してタスクの進捗を管理してください：
1. 各エージェントの作業をTaskCreateで登録
2. 依存関係をaddBlockedByで設定（Phase 2はPhase 1の全タスク完了後）
3. 作業開始時にin_progress、完了時にcompletedに更新

## 作業指示の共通事項

全エージェントに以下を指示してください：
- y/n確認は一切不要、直接作業を実施すること
- 作業完了後はチームリーダーにSendMessageで報告すること
- 日本語で回答すること

## あなた（チームリーダー）のタスク

1. TeamCreateでチーム作成
2. TaskCreateでタスク登録・依存関係設定
3. Phase 1: djns-scanner, ai-scanner, skill-scannerを**並列起動**
4. Phase 1完了後: 3エージェントの結果をreporterに共有してreporterを起動
5. reporterのレポートをユーザーに表示
6. チーム解散（TeamDelete）

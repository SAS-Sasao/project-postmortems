# DJNSプロジェクト - マスタ定義

課題の振り分け・分類に使用するマスタデータ。コマンド実行時に本ファイルを参照して自動分類を行う。

---

## 業務種別

| ID | 業務種別 | 説明 | 分類キーワード |
|----|----------|------|---------------|
| D01 | 要件定義 | 顧客要件のヒアリング・整理・要件定義書の作成 | 要件, 要求, ヒアリング, 仕様確認, スコープ, 機能一覧 |
| D02 | 設計 | 基本設計・詳細設計・DB設計・API設計 | 設計, DB, テーブル, API, 画面設計, ER図, クラス図 |
| D03 | 開発 | コーディング・実装・コードレビュー | 開発, 実装, コーディング, レビュー, プルリクエスト, バグ, 修正 |
| D04 | テスト | 単体テスト・結合テスト・総合テスト・受入テスト | テスト, 単体, 結合, 総合, 受入, 検証, 品質, 不具合 |
| D05 | 移行 | データ移行・環境構築・インフラ整備 | 移行, データ移行, 環境, インフラ, サーバー, デプロイ |
| D06 | リリース | 本番リリース・リリース判定・切替作業 | リリース, 本番, 切替, 公開, ローンチ |
| D07 | 運用 | 本番運用・障害対応・保守・問い合わせ対応 | 運用, 障害, 保守, 問い合わせ, 監視, パフォーマンス |
| D08 | PM・管理 | 進捗管理・リソース管理・ステークホルダー調整・WBS | PM, 管理, 進捗, スケジュール, 遅延, リソース, 会議, 調整, WBS |

## 影響度

| ID | レベル | 定義 | 分類キーワード |
|----|--------|------|---------------|
| H | 高 | プロジェクト全体の進行・品質・納期に重大な影響 | 致命的, 重大, 全体, 納期遅延, ブロッカー, 停止 |
| M | 中 | 特定の工程やチームに影響があるが全体は継続可能 | 一部, 特定, 工程, 遅れ, 手戻り |
| L | 低 | 軽微な影響、改善すればベター | 軽微, 小さい, 改善, ベター, 気になる |

## ステータス

| ID | ステータス | 定義 |
|----|-----------|------|
| S01 | 未着手 | まだ対応を開始していない |
| S02 | 進行中 | 対応を進めている |
| S03 | 完了 | 対応が完了し効果を確認済み |
| S04 | 保留 | 優先度や状況により一時保留 |

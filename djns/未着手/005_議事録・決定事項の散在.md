# 005: 議事録・決定事項の散在

- **日付**: 2026-03-07
- **業務種別**: PM・管理（ID: D08）
- **事実（Fact）**: 会議で決定した事項を取りまとめる議事録が複数箇所に点在し、後から事実を追うことが困難な状況になった
- **課題（Problem）**: 言った・言わないが発生し、事実確認のやり取りに相当数の工数を消費した
- **原因（Cause）**: 議事録・決定事項の記録場所と運用ルールが統一されておらず、各自が別々のツールや場所に記録していた
- **影響度**: 中（ID: M）
- **ステータス**: 未着手（ID: S01）

## 改善アクション

| # | アクション | 期限 | ステータス |
|---|-----------|------|-----------|
| 1 | 決定事項を一元管理するツール（Backlog等）を選定し、記録場所を最大2箇所に限定する運用ルールを策定する | - | 未着手 |
| 2 | 議事録テンプレート（日時・参加者・議題・決定事項・TODO・期限）を作成し、全会議で統一フォーマットを使用する | - | 未着手 |
| 3 | 会議終了後24時間以内に議事録を所定の場所に格納し、参加者全員に共有するルールを設ける | - | 未着手 |

## 次の打ち手（仮説）

- 記録場所を1〜2箇所に限定し議事録テンプレートを統一することで、「どこに何が書いてあるか分からない」状態を解消し、事実確認の工数を大幅に削減できるのではないか
- 決定事項にはBacklog等のチケット番号を紐づける運用にすることで、後から経緯を辿りやすくなり、言った・言わないの発生を防げるのではないか

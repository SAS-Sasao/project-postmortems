# 002: ステークホルダー要求のヒアリング不足

- **日付**: 2026-03-07
- **業務種別**: 要件定義（ID: D01）
- **事実（Fact）**: グループ会社3社でアプリケーションを統一しクラウド環境で運用したいという要望に対し、中心となる1社の現行販売管理システム機能を踏襲して機能追加するのか、完全新規で作成するのかを明確にしなかった
- **課題（Problem）**: ステークホルダーが求める結果の方向性が不明確なまま案件が進行し、取るべきアクションが定まらず後手に回った
- **原因（Cause）**: プロジェクト初期段階でステークホルダーへの深掘りヒアリングを怠り、要求の本質（踏襲 vs 新規）を確認しなかった
- **影響度**: 高（ID: H）
- **ステータス**: 未着手（ID: S01）

## 改善アクション

| # | アクション | 期限 | ステータス |
|---|-----------|------|-----------|
| 1 | プロジェクト初期に「既存踏襲 or 新規構築」等の根本方針を確定させるヒアリングシートを作成する | - | 未着手 |
| 2 | ステークホルダーへのヒアリング必須項目チェックリストを整備し、要件定義開始前に全項目を確認する運用とする | - | 未着手 |
| 3 | ヒアリング結果を文書化し、ステークホルダーの合意署名（または承認記録）を取得するプロセスを導入する | - | 未着手 |

## 次の打ち手（仮説）

- 要件定義フェーズ開始前に「スコープ確認ミーティング」を必須化し、方向性の合意を得てから着手することで、後工程での手戻りや判断遅延を防げるのではないか
- 複数ステークホルダーが存在する案件では、各社の優先度・期待値を個別にヒアリングした上で統合方針を決定する手順を設けることで、要求の抜け漏れを防げるのではないか

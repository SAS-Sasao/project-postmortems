# 006: PMによる意思決定の欠如

- **日付**: 2026-03-07
- **業務種別**: PM・管理（ID: D08）
- **事実（Fact）**: メンバーやリーダー間で意見が割れた際に、PMが最終責任を負い意思決定するという文化が存在しなかった。PM自身も一作業者として意見を言うのみで、何も決まらないという事象が数多く発生した
- **課題（Problem）**: 意思決定がなされないまま時間が経過し、プロジェクトの進行が停滞した
- **原因（Cause）**: PMの役割として「最終意思決定者」という責任と権限が明確に定義・共有されておらず、合議制のまま結論が出ない状態が常態化していた
- **影響度**: 高（ID: H）
- **ステータス**: 未着手（ID: S01）

## 改善アクション

| # | アクション | 期限 | ステータス |
|---|-----------|------|-----------|
| 1 | プロジェクト開始時に「PMが最終意思決定権を持つ」ことをチーム全員に明示し、役割分担表にも明記する | - | 未着手 |
| 2 | 意見が割れた場合の意思決定プロセス（下記参照）を定め、議論が一定時間で収束しない場合はPMが決定する運用とする | - | 未着手 |
| 3 | 意思決定の記録を議事録に「決定者・決定理由・決定日」として残し、後から経緯を追えるようにする | - | 未着手 |

### 意思決定プロセスの例

| ステップ | 内容 | 制限時間 |
|----------|------|----------|
| 1. 論点の明確化 | 何について決める必要があるかを整理 | 5分 |
| 2. 各自の意見表明 | メンバー全員が意見と根拠を述べる | 10分 |
| 3. 合意形成の試行 | 議論を通じて合意点を探る | 15分 |
| 4. PM判断 | 合意に至らない場合、PMが根拠を示した上で最終決定する | - |

## 次の打ち手（仮説）

- 意思決定プロセスを明文化し制限時間を設けることで、「議論が延々と続いて何も決まらない」状態を構造的に防げるのではないか
- PMが決定した理由を記録に残す運用にすることで、決定の透明性が担保され、チームの納得感と信頼関係を維持できるのではないか

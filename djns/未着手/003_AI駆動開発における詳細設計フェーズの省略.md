# 003: AI駆動開発における詳細設計フェーズの省略

- **日付**: 2026-03-07
- **業務種別**: 設計（ID: D02）
- **事実（Fact）**: AI駆動開発を前提としている案件にも関わらず、予算の関係上詳細設計書フェーズを省略した
- **課題（Problem）**: AIに読み込ませるために必要な設計情報・アーキテクチャ情報・記載ルールが不足し、AIをコントロールすることに却って時間を使ってしまった
- **原因（Cause）**: AI駆動開発においてAIが正しくコードを生成するために必要な設計情報の粒度を見極めず、予算削減の対象として詳細設計フェーズを安易に削ってしまった
- **影響度**: 高（ID: H）
- **ステータス**: 未着手（ID: S01）

## 改善アクション

| # | アクション | 期限 | ステータス |
|---|-----------|------|-----------|
| 1 | AI駆動開発に必要な最低限の設計ドキュメント一覧（下記参照）を定義し、予算削減時でも省略不可な項目を明確化する | - | 未着手 |
| 2 | 予算制約がある場合の「軽量設計書テンプレート」を作成し、フル版と軽量版の2段階で対応できるようにする | - | 未着手 |
| 3 | AI駆動開発の見積時に「AI向け設計ドキュメント作成工数」を必須項目として組み込む | - | 未着手 |

### AI駆動開発に最低限必要な設計情報の例

| 設計情報 | 内容 | 省略可否 |
|----------|------|----------|
| アーキテクチャ構成図 | システム全体の構成・技術スタック・通信フロー | 省略不可 |
| DB設計（ER図・テーブル定義） | テーブル構造・リレーション・制約 | 省略不可 |
| API仕様書 | エンドポイント・リクエスト/レスポンス定義 | 省略不可 |
| コーディングルール・命名規則 | AI生成コードの品質を統一するためのルール | 省略不可 |
| 画面設計（ワイヤーフレーム） | 画面遷移・レイアウト・入出力項目 | 軽量化可 |
| 業務ロジック定義 | 主要な業務フローと分岐条件 | 軽量化可 |

## 次の打ち手（仮説）

- AI駆動開発に必要な設計情報を事前に定義しておくことで、予算交渉の場で「ここは削れない」と根拠を持って説明でき、必要な設計工数を確保できるのではないか
- 軽量設計書テンプレートを整備しておくことで、予算制約下でもAIが正しく動作するための最低限の情報を担保できるのではないか

# 011: チームモチベーション醸成の不足

- **日付**: 2026-03-07
- **業務種別**: PM・管理（ID: D08）
- **事実（Fact）**: 本案件を達成することで各メンバーにどういったメリットや将来性が生まれるかなど、モチベーションに繋がる共有や説明がおろそかになっていた
- **課題（Problem）**: メンバーが「何のためにこのプロジェクトをやっているのか」を理解できず、主体的な推進力が生まれなかった
- **原因（Cause）**: PM自身がプロジェクトの意義・目的・メンバーへのメリットを腹落ちさせておらず、チームに対してビジョンを共有するプロセスが存在しなかった
- **影響度**: 中（ID: M）
- **ステータス**: 未着手（ID: S01）

## 改善アクション

| # | アクション | 期限 | ステータス |
|---|-----------|------|-----------|
| 1 | プロジェクト開始前にPM自身が「プロジェクトの目的・意義・各メンバーへのメリット」を整理するワークシート（下記参照）を作成し、自分自身に腹落ちさせる | - | 未着手 |
| 2 | キックオフミーティングで上記を全メンバーに共有し、「このプロジェクトを通じて何を得られるか」を明示する | - | 未着手 |
| 3 | 月次またはスプリント単位で振り返りの場を設け、達成した成果とメンバーの成長を可視化・共有する | - | 未着手 |

### プロジェクト意義整理ワークシート例

| # | 項目 | 内容 |
|---|------|------|
| 1 | プロジェクトの目的 | 何を実現するためのプロジェクトか |
| 2 | ビジネス上の価値 | 顧客・自社にとってどんな価値があるか |
| 3 | メンバーへのメリット | スキルアップ・キャリア・実績としての価値 |
| 4 | チームとしての成長機会 | 新技術の習得・チームワーク強化等 |
| 5 | 成功の定義 | 何をもって「成功」とするか |

## 次の打ち手（仮説）

- PMが自らプロジェクトの意義を言語化し共有することで、メンバーが「自分ごと」として捉え、主体的に動くチームを作れるのではないか
- 定期的に成果と成長を振り返る場を設けることで、モチベーションの低下を早期に検知し、プロジェクト中盤以降の士気低下を防げるのではないか

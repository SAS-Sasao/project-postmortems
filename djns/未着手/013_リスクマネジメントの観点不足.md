# 013: リスクマネジメントの観点不足

- **日付**: 2026-03-12
- **業務種別**: PM・管理（ID: D08）
- **事実（Fact）**: 事前にリスクを洗い出すことができず、急な作業やトラブルに対して後手に回ってしまった
- **課題（Problem）**: リスクマネジメントの観点が不足しており、発生し得るリスクへの事前対策が講じられていなかった
- **原因（Cause）**: プロジェクト開始時・各フェーズ開始時にリスクを洗い出し、対策を検討・管理するプロセスが確立されていなかった
- **影響度**: 高（ID: H）
- **ステータス**: 未着手（ID: S01）

## 改善アクション

| # | アクション | 期限 | ステータス |
|---|-----------|------|-----------|
| 1 | プロジェクト開始時にリスクアセスメントを実施するためのリスク洗い出しテンプレート（リスク項目・発生確率・影響度・対策・担当者）を作成する | - | 未着手 |
| 2 | 各フェーズ開始時にリスクレビューを実施し、リスク登録簿を更新する運用ルールを策定する | - | 未着手 |
| 3 | 週次または隔週でリスク状況を確認するチェックポイントを定例会議のアジェンダに組み込む | - | 未着手 |
| 4 | 過去案件で発生したトラブル・想定外事象を分類し、リスクカテゴリ別のチェックリスト（技術リスク・要件リスク・体制リスク・外部依存リスク等）を整備する | - | 未着手 |

## 次の打ち手（仮説）

- プロジェクト開始時にリスクアセスメントを必須プロセスとして組み込むことで、想定外のトラブルを事前に検知し、後手に回る状況を大幅に減らせるのではないか
- 過去案件のトラブル事例をリスクカテゴリ別に整理したチェックリストを活用することで、リスク洗い出しの精度が向上し、経験の浅いメンバーでも網羅的なリスク検討ができるようになるのではないか
